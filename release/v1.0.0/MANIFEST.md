# TahfidzFlow v1.0.0 — Release Manifest

This manifest lists **every file that belongs in the v1.0.0 release package** (the shippable
source distribution) and the rules for what is deliberately excluded. It is derived from the
Git-tracked contents at source commit `94ee97b`.

- **Total tracked files in repository:** 306
- **Files included in the release package:** 304 (all tracked files except the 2 legacy-bot
  files, which are optional — see below)
- **Deployable app root:** `web/` (Vercel Root Directory = `web`)

> The release is distributed as **source**, built on Vercel with `npm ci && npm run build`.
> There is no committed build output; Vercel produces `.next/` at deploy time.

---

## Include / exclude rules

**Include (ships in the package):**
- All application source under `web/src/`.
- Database schema, migrations, seed, and reset scripts under `web/prisma/`.
- Localization message catalogs under `web/messages/`.
- Static assets and PWA files under `web/public/`.
- Build/tooling config under `web/` (`package.json`, `package-lock.json`, `next.config.ts`,
  `tsconfig.json`, `eslint.config.mjs`, `postcss.config.mjs`, `vitest.config.ts`,
  `prisma.config.ts`, `.gitignore`, `.env.example`, `README.md`, `fix-env.js`).
- Repository-root config, CI, and documentation.

**Exclude (never ships / not committed):**
- `.env` and any real secrets — only `*.env.example` templates are included.
- `node_modules/` — restored via `npm ci`.
- `web/.next/`, `out/`, `build/`, `dist/`, `coverage/`, `.vercel/` — build/runtime output.
- `web/src/generated/prisma-next/` — the generated Prisma client (recreated by
  `prisma generate` / `postinstall`).
- `next-env.d.ts`, `*.tsbuildinfo` — regenerated by Next.js/TypeScript.
- OS/editor cruft (`.DS_Store`, `Thumbs.db`).
- `release/` package folder itself is documentation about the release, not part of the
  deployable app.

**Optional (project decision):**
- `legacy-bot/` — the previous Python Telegram bot (`tahfidz_app.py`, `requirements.txt`). It
  is not part of the v1.0.0 web application and can be excluded from a clean app-only archive.
  Listed at the end for completeness.

## How to reproduce this manifest

From the repository root at the release commit:

```bash
# Full tracked file list (should be 306)
git ls-files | wc -l

# App-only release package (excludes the 2 legacy-bot files -> 304)
git ls-files | grep -v '^legacy-bot/'

# Build a clean source archive of exactly the tracked files (respects .gitignore)
git archive --format=tar.gz --prefix=tahfidzflow-1.0.0/ -o tahfidzflow-1.0.0.tar.gz HEAD
```

Confirm the archive contains **no** `.env`, `node_modules/`, `.next/`, or
`web/src/generated/` entries before distributing.
